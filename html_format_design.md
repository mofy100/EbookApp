# 電子書籍アプリ専用 HTMLフォーマット仕様 (E-book DOM)

青空文庫のオリジナルHTML表記揺れを完全に吸収し、CSSによる「絶対的に安定した縦書きレイアウト」を保証するための、アプリ専用のクリーンなDOM構造を定義します。

## 1. 最上位構造
すべての作品は必ずこの構造にまとめられます。
```html
<article class="ebook-content">
  <header class="ebook-header">
    <h1 class="ebook-title">三十三の死</h1>
    <h2 class="ebook-author">素木しづ子</h2>
  </header>
  <div class="ebook-body">
    <!-- ここに本文要素が続く -->
  </div>
</article>
```

## 2. 本文要素のルール
すべてのテキストは必ず **ブロックレベル要素（`<p>`など）** にラップされます。
青空文庫のように「`<div>`直下にテキストノードがむき出しで、`<br />`で改行される」構造は全て撤廃し、CSSのグリッドや行間計算が完璧に機能するようにします。

### 2.1 通常の段落
```html
<p>　いつまで生きてていつ死ぬか解らない程、不安な淋しいことはないと、お<ruby>葉<rt>えふ</rt></ruby>は考へたのである。</p>
```

### 2.2 字下げ（インデント）と地付き
青空文庫の `margin-left` や `class="jisage_2"` は、すべて「意味論を持たせた専用のクラス」に変換します。これにより、CSS側で物理的なマージン干渉を起こさず、縦書き・横書き問わず安全に制御できます。

```html
<!-- 2文字の字下げ -->
<p class="indent-2">「これからなんでもお前の好きなことをしたがいい。」</p>

<!-- 地付き（行末揃え） -->
<p class="align-end">（大正三年十月）</p>
```

### 2.3 外字・画像の扱い
リンク切れを起こしていた相対ディレクトリ画像（`gaiji`など）は、パース時にアプリのローカルディレクトリに画像をダウンロードし、表示可能なAPIパスに書き換えます。
```html
<p><img src="/api/assets/gaiji/2-94-57.png" class="gaiji" alt="※(「土＋竒」、第3水準2-14-61)" /></p>
```

### 2.4 ルビの完全正規化
`<rp>` タグ（ルビ非対応環境用の括弧）はレイアウトノイズになるためパース時に完全に削除し、シンプルな `<ruby>` 構造に統一します。
```html
<ruby>漢字<rt>かんじ</rt></ruby>
```

### 2.5 縦中横（たてちゅうよこ）
全角・半角や不要なタグ付けを解消し、明確に `class="tatechuyoko"` に統一します。
```html
<span class="tatechuyoko">!!</span>
```

## 3. このフォーマットの圧倒的な優位性
1. **完全なCSSコントロール**: 全てが `<p>` ブロックベースになるため、今回問題になった「行間のガタつき」や「インデントの中途半端な干渉」がシステムレベルで発生しえなくなります。
2. **レスポンシブ完全対応**: `width`指定や物理マージンを取り除くことで、デバイスサイズや縦書き・横書きを自由にスイッチしてもレイアウトが一切破綻しません。
3. **高速＆クリーン**: APIがただのクリーンな静的HTMLを返すだけになるため、フロントエンドは「ただ表示するだけ」。不要な正規表現の除去等のオーバーヘッドがなくなります。
